Latency

Because a tally that lags is a tally people don't trust.

Measured p50 latencies across the Qesto path, from tap to projected bar. Edge-resident state means the wire distance is "to the nearest Cloudflare POP" — usually under 30ms.

Hop 01Client → edge POP. TLS to the nearest Cloudflare edge. Participant never talks to an origin.~22msp50 wire
Hop 02Edge → Durable Object. Session state lives in a single DO, pinned to the colo closest to first-mover.~8msintra-colo
Hop 03DO → WebSocket fan-out. Tally update broadcast to every connected participant over persistent WS.~12msfanout
Hop 04Render. Bar moves on the projected screen. Same pipe, no client poll.~16mspaint
TotalTap to bar movement. End-to-end, measured across 10k sessions in production.< 80msp50 end-to-end
Host controls

The console a facilitator actually wants.

Launch, pause, reopen

Every question is a state machine. Host controls all transitions; participants never see a broken flow.

Hide tally live

Toggle visibility mid-vote. Useful when you want the room to commit before seeing others' answers.

Option shuffle

Randomize order per participant. Defeats position bias without randomizing on the projected screen.

Minimum tally gate

Results stay hidden until N voters are in. Default 5, configurable. Prevents single-voter exposure.

Soft timer

Optional countdown visible to participants. Expires to close, not to lock — stragglers finish on next question.

One-click export

CSV, JSON, or signed PDF with every tally and consent log. Integrates with Workday, HRIS, Notion, Slack.
