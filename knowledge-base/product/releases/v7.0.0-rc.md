# v7.0.0-rc.1 — Release Candidate

**Status:** RC (staging)
**Sprint:** 97
**North star:** The network turn — federation (CONNECT) GA + privacy-native authoring (STUDIO),
proven at scale with zero cross-tenant leakage.
**GA target:** v7.0.0 at S99 (per [`SPRINT85_99_PLAN.md`](../planning/SPRINT85_99_PLAN.md)
§Release map — two-sprint RC: S97/S98 → S99 GA, with S98 reserved as the RC soak/harden +
DR-drill window).

This RC closes the E96 CONNECT and E97 STUDIO epics' high-risk surfaces and completes their
user-facing halves. CONNECT (cross-tenant federation) and STUDIO (AI-output authoring) were
deliberately sequenced across S95–S97 so their two highest-risk trust boundaries — *data
trust* (federation) and *AI-output trust* (authoring) — never co-landed at the same gate.

## Highlights

- **CONNECT federation GA.** The full federation lifecycle is feature-complete and proven at
  scale: scoped/TTL/HMAC-signed invites (`CONNECT-INVITE-01`, S95), federated join with
  invite-validity + revocation + tenant-match + region-match checks (`CONNECT-JOIN-01`, S96),
  aggregates-only cross-tenant views with an identity-leak guard (`CONNECT-ZEROK-01`, S96),
  region+tenant isolation reusing the `IsolationProof` harness (`CONNECT-ISOLATION-01`, S96),
  a hard sovereign-exclusion D1 `CHECK` constraint as the third structural layer
  (`CONNECT-SOVEREIGN-01`, S96), and jti-keyed invite revocation (`CONNECT-AUDIT-01`, S96). S97
  adds the federation UI (`FE-CONNECT-JOIN-UI-01`) so a participant can accept an invite and
  join through the product, seeing co-tenant stats as counts only — never names or identities.

- **`QA-CONNECT-SCALE-01` — 5-tenant × 50k × 100-query zero-leakage proof.** The release-blocking
  scale/isolation proof for ADR-0062: five tenants, 50,000 participants each, 100 cross-tenant
  aggregate queries per tenant, asserted with the existing `IsolationProof` +
  `aggregateIsSafe()` harnesses. Zero cross-tenant rows or identity leaks observed across the
  full run. This is the evidence gate for the v7.0 federation/isolation claim and unblocks
  `ADR-0062` §3/§5 sign-off.

- **STUDIO privacy-native authoring — complete authoring flow.** Workers-AI-only co-pilot
  generation (`STUDIO-COPILOT-01`, S96) and CANVAS theme-aware drafts (`STUDIO-THEME-01`, S96)
  are now wrapped in a full authoring UI (`FE-STUDIO-AUTHORING-01`): prompt input → theme
  selector → draft preview/edit → apply-to-session. `STUDIO-LIBRARY-01` adds save/list/fork —
  an operator can save an authored question, browse the library, and fork it into an
  independent, editable copy (same-tenant only this sprint; cross-tenant fork is deferred until
  after CONNECT GA stabilizes). `STUDIO-SUGGEST-01` adds semantic next-question suggestions via
  `DECISIONS_VECTORIZE`: after authoring one question, the UI offers a related second question.

- **`SEC-STUDIO-PROMPT-01` — prompt-injection hardening, Pentest #6 AI-safety scope closed.**
  `lib/studio-authoring.ts` and `lib/ai/prompt-sanitize` were hardened against an expanded
  injection corpus (instruction-override, role-play escape, delimiter-breakout, and nested
  payload variants). The eval gate (REV-10) carries updated golden fixtures in
  `tests/eval/fixtures/` covering accept-good / reject-malformed / reject-injection cases.
  This closes the AI-safety leg of Pentest #6 alongside the S95–S96 federation-trust and
  aggregation-isolation legs; **Pentest #6 overall crit/high = 0**.

- **Federation + authoring frontend and i18n.** New frontend surfaces for federation join and
  authoring (`FE-CONNECT-JOIN-UI-01`, `FE-STUDIO-AUTHORING-01`, library UI for
  `STUDIO-LIBRARY-01`) ship alongside `I18N-CONNECT-01` — federation labels and isolation
  warnings in all 5 locales (EN/NL/ES/DE/FR), `check:i18n` green.
