# v6.0.0 — GA

**Status:** GA
**Sprint:** 90
**Date:** 2026-06-19
**North star:** Platform v6.0 certification — the first major version since v5.0, certified
GA with the post-v5.0 expansion epics (TOWNHALL, STAGE, RETRO/IDEATE, DELIBERATE, EMBED,
CANVAS + CAPTIONS), a gov-cloud full-ATO **path**, a sovereign tenant tier, and bounded WCAG
AAA.

This promotes [`v6.0.0-rc.1`](./v6.0.0-rc.md) (S89) to GA. The RC was feature-complete; S90
is a **certification and release** sprint — no new feature GA, no new trust boundary.

## What ships at GA

- **v6.0 certification bundle — `PLATFORM-CERTIFICATION-V6-01` (ADR-0053).** A single
  queryable surface (`GET /api/platform/certification`) backed by the evidence index in
  [`PLATFORM_CERTIFICATION_V6.md`](../../security/PLATFORM_CERTIFICATION_V6.md): Pentest #5
  crit/high = 0, SOC 2 Type II + 2026 annual evidence, DR drill RTO ≤ 2h, bounded WCAG AAA,
  FedRAMP Moderate full-ATO **path**, and the sovereign tenant tier.

- **Platform version → `6.0.0`.** `GET /api/platform/version` reports `6.0.0`;
  `/api/platform/releases` adds `{ version: '6.0.0', codename: 'v6.0', status: 'ga',
  sprint: 90 }` (the RC entry stays in history).

- **v5.x deprecation policy — `V5X-SUNSET-NOTICE-01` (ADR-0053 §3).** v6.0 GA starts the
  v5.x deprecation clock and is **additive — no breaking API change vs. v5.x**. Surfaced at
  `GET /api/platform/v5-sunset` (`currentGa: '6.0.0'`, `v5MaintenanceEnd: '2028-12-31'`,
  policy doc → ADR-0053). No customer action is required; the notice exists so integrators
  can plan. Public API `v3` GA, `v2` maintained, `v1` deprecated (`Sunset-Date` headers).

- **Annual DR drill at GA cadence — `DR-DRILL-ANNUAL-V6-01`.** RTO ≤ 2h confirmed on the GA
  build; the S89 RC drill's open gaps (R2 snapshot cadence, D1 restore escalation) carried
  forward as **accepted residuals** with owners and targets, none GA-gating. See
  [`DR_DRILL_ANNUAL_V6_2026.md`](../../operations/DR_DRILL_ANNUAL_V6_2026.md). Next
  live-traffic drill: S98.

- **PEN5-E2 closed.** The one open RC architecture carry-forward (embed `tid`/`team_id`
  tenancy-model divergence) is **resolved by ratification** in
  [ADR-0050 Amendment 1](../../adr/ADR-0050-embeddable-sdk-auth-widget-origin-sandboxing.md):
  the embed tenancy key is intentionally the session owner's user id; isolation enforced
  fail-safe on the same value at both planes. No Medium remains open against GA.

## Honesty notes (claims stay bounded)

- **"Certified" = internal self-attestation bundle**, not a third-party-granted
  authorization. SOC 2 evidence is internal control-tracking; FedRAMP is a documented
  **path/target**, never an achieved ATO.
- **WCAG AAA is bounded** to core flows + captions overlay + canvas themes; the broader app
  is **AA** (`aaaConformance: 'partial'`). Do not say "AAA-conformant platform".
- All certification/compliance copy is gated by `npm run check:compliance-claims`.

## Quality gates

`tsc --noEmit` clean · Vitest green (incl. `tests/unit/platform-v6-ga.test.ts`) · AI eval
`npm run test:eval` green · `npm run build` green · Pentest #5 overall crit/high = 0.

## Upgrade

No action required. v6.0 is additive over v5.x — no breaking API change, no client migration.
v5.x enters maintenance (security + critical fixes) through 2028-12-31. Integrators on public
API `v1` should plan migration ahead of the `2027-12-31` sunset (already signalled via
`Sunset-Date` headers).
