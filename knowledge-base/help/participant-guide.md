# Joining and Participating in Sessions

## Do I Need an Account?

**No.** Participants join with a session link or short code. You do not need to sign up or install an app.

Hosts and team collaborators need accounts; voters typically do not.

## How to Join

Your host shares one of these:

1. **Session link** — click or paste in your browser
2. **Short join code** — enter at qesto.cc/join
3. **QR code** — scan with your phone camera

### Step by step

1. Open the link or enter the code
2. Enter a display name (unless the session is fully anonymous)
3. Accept GDPR consent if prompted (when the host collects identifiable data)
4. Wait for the host to open the session — you'll see energizers or the first question

Works on phone, tablet, and desktop. Use the latest version of Chrome, Firefox, Safari, or Edge.

## During the Session

- **Submit every answer** — tap or click **Submit**; a confirmation means your vote counted
- **Stay on the question** — once the host moves to the next question, late answers are not accepted
- **Watch the connection indicator** — if you see "Connection lost", refresh the page (F5)

## Can I Change My Answer?

It depends on the host's **vote policy** (set before the session starts):

| Policy | Behavior |
|--------|----------|
| **One vote** (default) | Submit once per question; changing is not allowed after submit |
| **Multiple votes** | Used for multi-select and upvote questions — select several options |
| **Reactions** | Live reaction-style voting; your latest choice replaces the previous one while the question is open |

If you submitted under "One vote" and need to correct something, ask the host — they cannot edit your response, but may note it verbally or re-ask.

## Anonymous vs Named Sessions

- **Named** — your display name may appear to the host (not always to other participants)
- **Anonymous** — responses are not attributed to you
- **Partial / zero-knowledge** — stricter anonymity modes for sensitive feedback

The host chooses this before going live. You cannot switch modes yourself.

## After the Session

When the host closes the session:

- You may see a thank-you or summary screen
- Final results depend on what the host shares — some sessions show live results, others only on the host's screen
- You cannot rejoin a closed session

## Troubleshooting for Participants

| Problem | Fix |
|---------|-----|
| Link says session not found | Host may not have started yet — ask them to click **Start** |
| "Participant limit reached" | Host's plan is full — they need to upgrade or run a second session |
| My vote didn't count | Did you click Submit? Refresh and check the question is still open |
| Page frozen | Refresh (F5); your submitted answers are saved server-side |

More detail: see the Troubleshooting guide or email support@qesto.cc with the session code and a screenshot.
