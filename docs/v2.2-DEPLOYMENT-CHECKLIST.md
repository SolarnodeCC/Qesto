**This is an early v2.2 release due to accidental schema application to production.**

**What's Safe:**
- ✓ Circuit breaker, PII sanitization, integration libraries are production-ready
- ✓ Feature flags OFF means v2.2 UI features won't activate
- ✓ Database schema exists but no code uses it yet (safe state)
- ✓ No production data corruption risk

**What's NOT Activated:**
- LIVE energizers (Quick Finger, Team Quiz) — behind LIVE_ENERGIZERS_ENABLED flag

## Deployment Steps (Run on Your Local Machine)

### Step 1: Switch to main and pull latest

```bash
git checkout main
git pull origin main
```

### Step 2: Verify build succeeds

```bash
npm run build
```

**Expected:** ✓ built in ~5s, dist/ folder created

### Step 3: Deploy to production

```bash
wrangler pages deploy dist --project-name=qesto
```

**Expected output:**
```
✓ Uploading files [====================] 100%
✓ Deployment successful
✓ Uploaded 147 files (234 KB)
✓ Deployment URL: https://qesto.cc/
```

**What happens:**
1. Static frontend deployed to Cloudflare Pages
2. Worker API (functions/api/[[route]].ts) with v2.2 code deployed
3. Circuit breaker, PII sanitization, integration libraries loaded
4. Feature flags OFF = v2.2 routes exist but disabled

### Step 4: Verify deployment (5 min post-deploy)

```bash
# Health check
curl https://qesto.cc/api/admin/health

# Version check
curl https://qesto.cc/api/version

# Expected: HTTP 200 with JSON response
```

### Step 5: Monitor logs (10-15 minutes)

```bash
wrangler tail qesto --project-name=qesto
```

**Watch for:**
- Error rate stable (<0.1%)
- No circuit breaker triggers (should be 0)
- Normal request volume
- No PII in logs (verify with grep "REDACTED")
