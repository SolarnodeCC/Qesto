# Qesto v0.1.0 Release Notes

**TL;DR**: Qesto v0.1 is the first shippable greenfield release. You can now create and run live interactive sessions with polls, rankings, and consent votes — with real-time WebSocket delivery, instant results export, and privacy built in.

---

## What's New

### Core Platform
- **Live Session Engine**: Create sessions in DRAFT, transition to LIVE with a single click. Real-time WebSocket keeps all participants synced within 100ms.
- **Real-time Voting**: Submit poll votes, rankings, and consent responses. Your vote is recorded instantly and broadcast to other participants.
- **Four Question Types**: Polls (2–5 options), rankings (2–10 items), consent votes (Agree/Neutral/Disagree), and open questions with optional character limits.
- **Presenter Controls**: Advance questions, show/hide results, pause voting — all in real-time with zero lag.
- **Participant Join Flow**: Share a session link with a unique code. Participants join via shareable URL or code entry; anonymous participation supported.

### Authentication
- **Magic Link Login**: No passwords. Enter your email, click the link in your inbox, set a display name. JWT-based sessions work across all browsers.
- **Team Ownership & Sharing**: Create teams, invite others, and co-host sessions. Session ownership enforced at the data layer.

### Results & Export
- **Live Results Dashboard**: After a session closes, view vote counts, rankings, and open responses in a clean results page.
- **CSV Export**: Download all votes and responses as a single CSV file for offline analysis. Columns include question text, answer, participant name, and timestamp.

### Platform
- **Structured Logging**: All API calls and WebSocket events are logged with request ID, status, and latency for debugging.
- **Health Endpoint**: `/api/health` confirms the system is ready to serve requests.
- **KV Rate Limiting**: Protects API routes from abuse with token-bucket rate limits per team.

### Accessibility
- **WCAG 2.1 Foundation**: Page structure, link labels, and form elements follow WCAG 2.1 AA. Keyboard navigation works on join and results pages.
- **Color Contrast**: Primary CTAs and results charts meet 4.5:1 contrast ratio (AA standard).

### Internationalization
- **Locale Bundles Ready**: English is the shipped language. Scaffolding for Dutch (NL), Spanish (ES), German (DE), and French (FR) is in place; translations pending.
- **Browser Language Detection**: App detects your preferred language from browser settings (future: language selector UI).

---

## Known Limitations

- **Single Question per Session**: Sessions support one active question at a time. Multi-question branching, energizers (speed rounds, trivia), and leaderboards ship in later releases.
- **No Billing Integration**: All teams are on Free tier; Stripe checkout and subscription management are not yet live.
- **No SAML SSO**: Enterprise single sign-on via Okta, Azure AD, or similar is planned for Phase 3.
- **No Multi-Language Content**: Only English content ships; translations for NL/ES/DE/FR are prepared but not yet live.
- **No Stress Testing Above 25 Concurrent**: Current load test target is 25 simultaneous participants per session. 100+ concurrent testing is planned for Phase 5.
- **No AI Insights**: Workers AI endpoints exist in the stack; AI-powered question suggestions and result summaries ship in Phase 6.

---

## Upgrade Path

This is v0.1.0 (first release). No migration path applies.

---

## Get Started

1. **Create a Session**: Log in, create a new session, add questions (poll, ranking, etc.), and save.
2. **Go Live**: Click "Start" to transition to LIVE. Your session link is ready to share.
3. **Participants Join**: Share the URL or code with attendees. They join with their name (or anonymously).
4. **Run the Session**: Advance questions, show results, manage timing.
5. **Export Results**: After closing, download a CSV with all votes.
